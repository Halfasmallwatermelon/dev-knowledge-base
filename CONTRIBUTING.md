# 贡献指南

感谢你对本仓库的关注！这是一个面向开发者的**知识学习与沉淀**开源项目，欢迎通过 Issue 与 Pull Request 参与共建。

## 你可以如何贡献

| 方式 | 适用场景 |
|------|----------|
| 提交 Issue | 发现错误、链接失效、概念表述不清、希望新增某主题 |
| 提交 Pull Request | 直接修正文档、补充示例、新增笔记 |
| 分享与 Star | 帮助更多人发现本项目 |

## 开始之前

1. 浏览 [README.md](README.md) 了解目录结构与使用方式
2. 阅读 [AGENTS.md](AGENTS.md) 了解 Cursor Agent 规则与技能
3. 新增笔记请参考 [笔记模板](.cursor/skills/write-learning-note/template.md)

## 提交 Issue

请尽量说明：

- **位置**：文件路径与章节（如 `database/mysql-index.md` 第 3 节）
- **问题**：错误描述、期望内容或建议主题
- **依据**：官方文档、RFC、规范等参考链接（如有）

可使用 `.github/ISSUE_TEMPLATE/` 下的模板快速填写。

## 提交 Pull Request

### 流程

1. Fork 本仓库
2. 从 `main` 创建功能分支，命名建议：
   - `fix/描述` — 修正错误
   - `docs/领域-主题` — 新增或完善笔记
3. 按规范修改或新增 Markdown 文件
4. 提交 PR 并填写模板中的说明项
5. 等待 Review 与合并

### 笔记规范

单篇笔记推荐结构：

1. 概述（解决什么问题）
2. 核心概念
3. 典型用法 / 示例
4. 常见误区与最佳实践
5. 与其他领域的关联
6. 参考资源（**优先官方文档与标准规范**）

写作原则：

- **由浅入深**：先概念与动机，再原理，最后实践
- **可验证**：提供可运行的最小示例、命令或检查清单
- **可检索**：包含「关键词」「前置知识」「延伸阅读」
- **中文为主**：术语首次出现附英文原文

### 提交信息

建议使用清晰的中文或英文 commit message，例如：

```
docs(database): 补充 MySQL 联合索引最左前缀说明
fix(frontend): 修正 useEffect 依赖数组示例
```

### 注意事项

- 不要提交密钥、`.env`、私钥等敏感信息
- 避免大段复制受版权保护的内容；引用请注明出处
- 单 PR 聚焦一个主题，便于 Review
