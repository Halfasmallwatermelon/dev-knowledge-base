# 编写学习笔记

## 工作流

1. **检索**：Glob/Grep 同目录已有文档，避免重复
2. **定位**：按主题选目录（见 AGENTS.md 领域表）
3. **命名**：`主题-子主题.md`，英文 kebab-case 或中文，保持同目录一致
4. **写作**：套用 [template.md](template.md)
5. **关联**：文末链到相关笔记；跨领域时注明上下游关系

## 零基础写作要求

- 默认读者**没有编程基础**，禁止跳步、禁止裸堆术语
- 代码/命令**必须带注释**；代码块下方写「逐步讲解」
- 核心概念表含「通俗解释」列（类比或人话）
- 新语法单独说明「这是什么、为什么这样写」

## 质量检查

- [ ] 有最小可运行示例或命令，且带注释
- [ ] 示例下方有逐步讲解（这段代码做了什么）
- [ ] 术语首次出现附英文 + 一句话人话解释
- [ ] 含「常见误区」或「踩坑」（针对新手易错点）
- [ ] 参考资源以官方文档为首
- [ ] 复杂流程有 mermaid 图

## 增量更新

编辑已有文档时：
- 保留原有结构，在合适章节追加
- 重大变更在文首加「更新记录」
- 不删除用户已有笔记内容，除非明显错误且需标注修正原因

## 领域规则

写作时对应 `.cursor/rules/*-learning.mdc` 会自动生效；也可主动 Read 该规则以对齐领域重点。
