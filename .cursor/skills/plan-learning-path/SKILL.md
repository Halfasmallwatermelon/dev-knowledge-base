# 规划学习路径

## 工作流

1. **摸底**：问清或推断当前水平、目标（就业/项目/面试/兴趣）、时间预算
2. **拆解**：按「基础 → 核心 → 实战 → 进阶」分阶段，每阶段 1–4 周可完成
3. **串联**：标注跨领域依赖（如 HTTP 同时服务前后端）
4. **产出**：写入 `general/learning-paths/` 或直接在对话中给出

## 路径模板

```markdown
# [目标] 学习路径

## 目标与前提
- 目标：…
- 前提：…
- 预计周期：…

## 阶段一：[名称]（第 N 周）
**目标**：…
**知识点**：…
**实践**：…
**验收**：能独立完成 …
**笔记目录**：`frontend/xxx.md`

## 阶段二：…

## 里程碑检查清单
- [ ] …

## 推荐资源
- …
```

## 默认全栈骨架（可按需裁剪）

| 顺序 | 领域 | 核心内容 |
|------|------|----------|
| 1 | general | Git、命令行、HTTP 基础 |
| 2 | frontend | HTML/CSS/JS → 框架入门 |
| 3 | backend | REST API、分层架构 |
| 4 | database | SQL、建模、索引 |
| 5 | testing | 单元测试 → 集成测试 |
| 6 | deployment | Docker → CI/CD → 云部署 |

## 原则

- **默认零基础**：路径从「完全不会写代码」出发，每阶段标注「你需要先懂什么」
- 每阶段必须有**可验收的动手项目**，不单列阅读清单
- 优先官方文档 + 本仓库对应目录笔记
- 路径过长时分「最小可用路径」与「扩展路径」
